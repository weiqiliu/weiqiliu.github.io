#Android Metarial Design Component

Android 5.0发布了Metarial design设计规范，也发布了许多新组件，在这里总结一些比较好的，未来项目可能会用到的组件的介绍和基本用法。

  • ###ToolBar

    先给个不错的链接介绍
    google推荐的App bar标准，但是有些缺陷，这个bar的高度为actionBarSize
    内部的所有图标文字均是按这个高度来设计的，所以有了一定的局限性，可能是Google为了规范标准吧。它规定了返回图标，logo，主副标题，更多图标。

  • ###AppBarLayout

    AppBarLayout的作用就是把多个component组装成一个AppBar，通常这个组件是配合后面介绍的 CoordinatorLayout 和 CollapsingToolbarLayout来使用了，另外它提供的OnOffsetChangedListener的接口方法使我们很容易的获取我们的滑动状态

  • ###CoordinatorLayout

    首先你可以把coordinatorLayout当做一个可以滚动的FrametLayout,一样提供诸如android:layout_gravity等设置参数,它的布局方式默认也是一层一层叠上去得。它是用来协调它的子views之间协作的一个父view。这和传统的ViewGroup不同，子view从此知道了彼此之间的存在，一个子view的变化可以通知到另一个子view。CoordinatorLayout所做的事情就是当成一个通信的桥梁，连接不同的view。使用Behavior对象进行通信。

注意里面的

1.app:layout_behavior=”@stringappbar_scrolling_view_behavior”
这个就是设置了behavior参数，使这个recyclerview与Appbar关联了起来.
2.app:layout_scrollFlags这个参数将设置滑动的效果

– scroll: this flag should be set for all views that want to scroll off the screen - for views that do not use this flag, they’ll remain pinned to the top of the screen
– enterAlways: this flag ensures that any downward scroll will cause this view to become visible, enabling the ‘quick return’ pattern
– enterAlwaysCollapsed: When your view has declared a minHeight and you use this flag, your View will only enter at its minimum height (i.e., ‘collapsed’), only re-expanding to its full height when the scrolling view has reached it’s top.
– exitUntilCollapsed: this flag causes the view to scroll off until it is ‘collapsed’ (its minHeight) before exiting(向上滚动时收缩View，但可以固定Toolbar一直在上面。)

后两种要配合下面介绍的Collapsingtoolbarlayout才会生效
