Android SQLite

Android SQLite

1.SQLite简介

SQLite是一款轻型的数据库，是遵守ACID的关联式数据库管理系统，它的设计目标是嵌入 式的，而且目前已经在很多嵌入式产品中使用了它，它占用资源非常的低，在嵌入式设备中，可能只需要几百K的内存就够了。它能够支持 Windows/Linux/Unix等等主流的操作系统，同时能够跟很多程序语言相结合，比如Tcl、PHP、Java、C++、.Net等，还有ODBC接口，同样比起 Mysql、PostgreSQL这两款开源世界著名的数据库管理系统来讲，它的处理速度比他们都快。

2.SQLite数据类型

一般数据采用的固定的静态数据类型，而SQLite采用的是动态数据类型，会根据存入值自动判断。SQLite具有以下五种常用的数据类型：

NULL: 这个值为空值
VARCHAR(n)：长度不固定且其最大长度为 n 的字串，n不能超过 4000。
CHAR(n)：长度固定为n的字串，n不能超过 254。
INTEGER: 值被标识为整数,依据值的大小可以依次被存储为1,2,3,4,5,6,7,8.
REAL: 所有值都是浮动的数值,被存储为8字节的IEEE浮动标记序号.
TEXT: 值为文本字符串,使用数据库编码存储(TUTF-8, UTF-16BE or UTF-16-LE).
BLOB: 值是BLOB数据块，以输入的数据格式进行存储。如何输入就如何存储,不改 变格式。
DATA ：包含了 年份、月份、日期。
TIME： 包含了 小时、分钟、秒。

3.SQLite注意事项（重点）

  • 库级锁：意味着同时只能允许一个写操作，也就是说，即事务T1在A表插入一条数据，事务T2在B表中插入一条数据，这两个操作不能同时进行，即使你的机器有100个CPU，也无法同时进行，而只能顺序进行。表级都不能并行，更别说元组级了——这就是库级锁。对于这样的问题，解决的办法就是keep single sqlite connection，保持SqliteOpenHelper是单例模式，同时对所有数据库操作的方法添加synchronized关键字。
  • 事务：使用事务可以有效的提升的性能

4.SQLiteOpenHelper
Android平台提供给我们一个数据库辅助类来创建或打开数据库，这个辅助类继承自SQLiteOpenHelper类

利用这个帮助类能很好的创建，更新数据库。其中的onUpgrade()只有当版本号不同时才会执行，当表有更新时通过改变版本号来重新建立表。 另外由于可能会在许多地方利用数据库，为了避免内存泄露和不必要重复分配。最好的办法就是SQLiteOpenHelper采用单列模式(singleton instance)。

5.SQLite的query、insert、update、delete
这四种操作都有两种方法可以达到目的，第一种是采用Android提供的方法，第二种是直接执行sql语句。

  • insert:
    public long insert (String table, String nullColumnHack, ContentValues values)

    table:表名。
    nullColumnHack:当values参数为空或者里面没有内容的时候，我们insert是会失败的（底层数据库不允许插入一个空行），为了防止这种情况，我们要在这里指定一个 列名，到时候如果发现将要插入的行为空行时，就会将你指定的这个列名的值设为null，然后再向数据库中插入。
    values:一个ContentValues对象，类似一个map.通过键值对的形式存储值。

  • update:

    public int update (String table, ContentValues values, String whereClause, String[] whereArgs)
    values:要更新的数据内容
    whereClause：更新的字段名称,为null即为全部字段
    whereArgs：whereClause里的占位符

  • delete:

public int delete (String table, String whereClause, String[] whereArgs)

  • query

public Cursor query (String table, String[] columns, String selection, String[] selectionArgs, String groupBy, String having, String orderBy,String limit)

columns:想要显示的列，若为空则返回所有列，不建议设置为空.
selection:where子句，声明要返回的行的要求，如果为空则返回表的所有行.
selectionArgs: selection:where的占位符
groupBy: 分组方式，若为空则不分组.
having : having条件，若为空则返回全部（不建议)
orderBy: 排序方式，为空则为默认排序方式.
limit:限制返回的记录的条数，为空则不限制.

query返回的是个Cursor。然后通过Cursor可以获取查询出来的内容
